# Frontend UI Checklist

## Hierarchy

- Make the main task visible immediately.
- Group related controls together.
- Reduce competing calls to action.

## Forms

- Label every field clearly.
- Explain disabled states.
- Preserve inputs on validation failures.
- Keep submit actions specific: `Create classroom`, `Save level`, `Open editor`.

## Editor UX

- Keep the tool legend near the editor.
- Explain drag, place, and erase behavior.
- Distinguish quick-edit tools from full-edit tools.
- Show a summary of what has been placed.

## State Design

- Add useful empty states.
- Add visible success states after save or completion.
- Show completed states instead of silently replaying a flow.

## Responsiveness

- Ensure primary actions remain visible on smaller screens.
- Collapse multi-column layouts cleanly to one column.
- Avoid pushing critical content below large decorative blocks.
